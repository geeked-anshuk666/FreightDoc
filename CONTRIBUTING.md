# Contributing to FreightDoc

Thank you for your interest in contributing to FreightDoc! Please follow these guidelines when submitting code or documentation updates.

## Code of Conduct
Please maintain a professional, respectful environment in all project communications and contributions.

## Development Process
1. **Fork the Repository:** Create a personal fork of the repository to work on.
2. **Create a Feature Branch:** Branch off from `main` with a descriptive name (e.g., `feat/add-new-corridor` or `fix/validation-logic`).
3. **Local Setup:** Follow the environment setup instructions in the `README.md`.
4. **Testing:** Run existing tests (`pytest` for backend, `npm run test` for frontend) to ensure no regressions are introduced. Write new tests for any newly added features.
5. **Commit Conventions:** Follow clear, descriptive commit messages.
6. **Submit a Pull Request:** Open a PR against the upstream `main` branch with a summary of changes, rationale, and verification details.

## Documentation Guidelines
Keep documentation concise and accurate. Ensure that any architectural changes are reflected in the corresponding diagrams. Do not include sensitive information or private credentials in the documentation.
