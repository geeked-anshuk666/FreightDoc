# FreightDoc competitor research framework

## Status

This document is a research plan, not an assertion of current competitor features, pricing, traffic, customers, or rankings. The product is new and this repository does not contain verified live competitive research. Validate every observation against the competitor's current public site and record the date before publishing comparison content.

## Candidate research set

The following are candidate categories to investigate because they may overlap with trade operations, customs, freight forwarding, or documentation workflows. Their inclusion is not a feature comparison or endorsement.

| Candidate | Why research it | Verification required before any public claim |
| --- | --- | --- |
| Descartes | Trade and logistics software category | Current product scope, geography, buyer segment, current pages |
| CargoWise | Freight-forwarding operations category | Current modules, target users, integrations, public terminology |
| e2open | Supply-chain software category | Current trade/compliance positioning and documentation coverage |
| WiseTech Global | Logistics technology category | Current portfolio and audience segmentation |
| Avalara | Tax/compliance software category | Whether a specific page overlaps with customs or trade-documentation intent |

## Research method

For each candidate, capture only verifiable public material:

1. Home, product, documentation, security, pricing, and resource URLs.
2. The exact query and search-result date used to discover each page.
3. Visible content format: guide, calculator, landing page, documentation, comparison, webinar, or case study.
4. Structured-data types observed in source, performance signals from public tools, and accessibility findings.
5. Keyword themes and unanswered questions. Do not estimate traffic or domain authority without a licensed, current data source.
6. A legal review of any future comparison page for nominative use, source citations, and update date.

## Opportunity hypotheses to test

- Explain the exporter-to-broker handoff in plainer language than generic supply-chain software pages.
- Build source-dated, corridor-specific preparation guides rather than unsupported “instant compliance” claims.
- Publish transparent data-retention and document-intake safety information.
- Help users distinguish trade-statistics context from live duty evidence.

These are hypotheses, not conclusions. They require keyword research, real user interviews, and source validation before prioritization.

## Comparison-page rule

Do not create `FreightDoc vs. <brand>` pages until FreightDoc has an accurate, reviewable feature matrix and a dated source archive for the other product. Comparison pages must explain differences without implying affiliation, copying brand assets, or making unverified pricing, performance, or legal claims.
