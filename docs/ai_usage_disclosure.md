# AI usage disclosure

## Product runtime

FreightDoc is deterministic-first. Country rules, document requirements, validation paths, records, PDFs, and manual review remain usable without an AI key. Optional runtime suggestions are configured separately through `AI_PROVIDER`/`AI_MODEL` (the current executable defaults are Groq-oriented); they are not a claim about development tools and require human acceptance.

## Development-tool evidence gate

The Build Week submission requires Codex and GPT-5.6 use. Add a public or spoken claim only after recording, in permitted local evidence:

- exact displayed tool/model name and date;
- verified work assisted;
- displayed effort level, only if the record actually displays one;
- the Codex `/feedback` Session ID for the Devpost form.

Do not infer a ChatGPT model variant or effort level. Do not publish a raw session transcript, prompts containing private data, credentials, or the session ID unless the submission form specifically requires it.

### Approved disclosure slot

`Codex: [verified work]. GPT-5.6: [verified work]. [State an effort level only when the corresponding local record displays it.] Runtime suggestions are optional and separately configured.`

Before final submission, replace bracketed text only with verified facts; otherwise use: `Development-tool details are recorded in the submission evidence; runtime suggestions remain optional and human reviewed.`
