# FreightDoc — risks and safeguards

| Risk | Current safeguard |
|---|---|
| Incorrect classification or requirements | Suggestions and fallbacks are labelled; human review is required; no legal or clearance guarantee. |
| Changing rules or sources | Versioned local rules, provenance, and a stated need for qualified verification. |
| Optional AI/provider failure | Deterministic/manual workflow remains usable without an AI key. |
| Unsafe document intake | Bounded file/type/resource checks and transient original bytes. |
| Mock connector confusion | Manual/mock interfaces must never claim filed, cleared, screened, accepted, or certified. |
| Demo infrastructure failure | Pre-warm services and show the deterministic/manual path honestly. |

Free-tier hosting has cold-start and quota limits; it is not an SLA-backed production service.
